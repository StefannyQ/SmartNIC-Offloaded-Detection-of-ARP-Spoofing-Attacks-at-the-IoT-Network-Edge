# Replication package

Code for the ARP-spoofing-detection / SmartNIC-deployability pipeline used
in the paper: data cleaning → EDA → train-only feature engineering and
model training under SmartNIC hardware constraints (S1-S4 scenarios) →
integer quantization → P4/BMv2 hardware-fidelity validation.

No data or trained models are included here -- only the code. The dataset
(CICIoT2023) cannot be redistributed under its usage terms, and the
trained models are multi-gigabyte binaries that don't belong in a git
repo; both are regenerated by running the pipeline.

## Pipeline

| Script | Stage | Output |
|---|---|---|
| `01_data_cleaning/01_data_cleaning.py` | Load raw CICFlowMeter CSVs, apply domain-grounded cleaning fixes, validate against impossible values, remove outliers/duplicates | `data/final_dataset.csv` |
| `02_eda/02_eda.py` | Purely descriptive EDA on the full cleaned dataset: distribution shape, class balance, unsupervised structure (PCA/K-Means/t-SNE). No label-dependent feature decisions -- those require a train split, done in stage 3 | `plots/*.png` |
| `03_modeling/03_modeling.py` | Train/val/test split; train-only statistical feature reduction; SmartNIC-observable feature subsetting; engineered-feature screening; trains 4 scenarios (S1 Gateway Baseline, S2 SmartNIC Observable, S3 SmartNIC Deployable, S4 SmartNIC Deployable + FE); derives and verifies integer quantization for S2-S4; threshold analysis; feature importance; lift/cumulative-gains analysis | `modeling/{results,models,plots}/*` |
| `04_p4_validation/04_generate_p4.py` | Extracts the fitted, integer-quantized S4 Decision Tree and generates a P4_16 program plus a Scapy test harness that validate the tree's match-action logic against BMv2 | `p4_validation/*` |

Each stage reads the previous stage's checkpoint and writes its own; run
them in order. See `outputs/README.md` for exactly what every generated
file contains.

## Reproducibility

- **Random seed**: `random_state=42`, fixed everywhere a `scikit-learn`
  call accepts one (the train/val/test split, every model, every
  stratified sample). This is set once per script and never overridden.
- **Library versions**: pinned exactly (not `>=`) in `requirements.txt`.
  `scikit-learn`'s CART split selection and tie-breaking are
  version-sensitive enough to shift the exact tree learned -- install the
  pinned versions, not just "recent" ones, if you need the same tree
  structure reported in the paper.
- **Python**: tested with 3.14.2.

## Running the pipeline

From the repo root, with `data/{Benign_Final,MITM-ArpSpoofing}/` already
in place (see `data/README.md`):

```bash
python 01_data_cleaning/01_data_cleaning.py   # -> data/final_dataset.csv
python 02_eda/02_eda.py                       # -> plots/
python 03_modeling/03_modeling.py             # -> modeling/{results,models,plots}/
python 04_p4_validation/04_generate_p4.py     # -> p4_validation/
```

`03_modeling.py` is the expensive stage (full hyperparameter search across
4 scenarios, plus integer retraining); the others run in minutes.

Every script resolves its own paths relative to its own file location
(`repo_root = parent of the folder containing the script`), so the repo
works regardless of where it's cloned -- no paths need editing.

## P4 hardware-fidelity validation

`04_generate_p4.py` only generates the P4 program and test harness; it
doesn't compile or run them -- that needs `p4c` and BMv2 (`simple_switch`)
on a Linux environment (a VM is the simplest route). Full setup, exact
tool versions, and the actual measured result (**99.16% exact match, zero
mismatches** against the Python model on the full 210,863-row test set)
are in `04_p4_validation/vm_setup.md`.
